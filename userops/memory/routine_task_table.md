# ROUTINE TASK TABLE

Routine checks are configurable defaults, not immutable law.

| Check | Default frequency | Trigger | Action | Status |
|---|---:|---|---|---|
| Unclosed task state | every task close | task folder lacks closure note | draft closure summary | active |
| Stale contract before execution | configurable | delay between contract and execution | suggest preflight | active |
| Missing reviewer record | every staged task | review happened but no log exists | ask for reviewer record | active |
| Memory candidates review | every task close | reusable lesson appears | add to memory candidates | active |
| Trap archive review | every repeated failure | same avoidable mistake appears twice | record trap candidate | active |
| Council re-entry check | each evidence reversal | execution evidence breaks assumptions | prepare re-entry package | active |
| Public-safety scan | before public release | files move from private to public | scan for names, paths, secrets, and non-public text | active |

## Notes

Routine checks should not interrupt urgent execution unless they affect the current task's safety, authority, or traceability.

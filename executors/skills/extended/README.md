# Extended Skills

`extended/` holds specialised branches for narrower domains.

Current live branches:

- `development/` for coding, web, plugin, MCP, and engineering workflows
- `design/` for creative direction, visual systems, themes, and brand-facing design work
- `communication/` for writing, coauthoring, and internal communications workflows
- `research/` for specialised analysis, audit, investigation, and reasoning-heavy domains

Vetted third-party imports should not form a parallel skill class.
They belong directly inside the branch they semantically fit.
Track provenance in [`executors/skills/IMPORTS.md`](../IMPORTS.md), not in extra wrapper path levels.

Priority rule:

- formal Council contract beats skill workflow on conflict
- skills help with method, not mandate expansion of scope
- if a skill suggests actions that would violate a frozen boundary, file scope, or forbidden action in `CORE_*` / `EXT_*`, follow the contract and ignore that part of the skill

For the 2026-03-23 Anthropic intake, no imported skill cleared the `core/` bar.
They were all useful, but still too domain-specific to count as base executor method.
